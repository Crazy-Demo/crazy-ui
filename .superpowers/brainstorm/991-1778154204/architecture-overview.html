企业级 UI 组件库推荐架构：Core + Vue Adapter

第一阶段落地 Vue 3，底层能力保持框架无关，为 React / Web Components 预留扩展空间。

分层架构总览
生态层 Ecosystem
Docs / Playground / CLI / Resolver / Figma Tokens / ESLint Rules
业务层 Pro Components
ProForm / ProTable / QueryFilter / Layout / Auth-aware Widgets
组件层 Vue Components
Button / Input / Select / Dialog / Form / Table / DatePicker
适配层 Vue Adapter
Plugin Install / Directives / Composables / Runtime Config / Locale Provider
基础层 Framework-agnostic Core
Tokens / Theme Algorithm / I18n Protocol / Icon Model / Shared Utils / Types
样式层 Style System
SCSS Source / CSS Variables / Theme Packages / Runtime Theme Injection
工具层 Tooling
Build / Test / Release / Type Check / Bundle Analyze / Codemod

核心原则

  • 业务只能依赖组件，组件只能依赖 Core、样式和工具，Core 不依赖任何框架。
  • 主题、图标、国际化、类型协议沉淀到 Core，Vue 只做渲染和运行时适配。
  • 每个组件独立入口、独立样式、独立类型，保证按需引入和 Tree-shaking。
